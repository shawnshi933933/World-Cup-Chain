# 炜新联手征战世界杯 — 项目总结

> 个人串关投注辅助工具，对接 Polymarket 2026 FIFA 世界杯预测市场。  
> 当前状态：**已部署上线**，仿真模式完全可用，真实下单受 Polymarket 地区封锁限制（见末节）。

---

## 一、整体架构

```
pnpm monorepo (Replit)
├── artifacts/world-cup/        # 前端 React + Vite
├── artifacts/api-server/       # 后端 Express 5
└── lib/
    ├── db/                     # Drizzle ORM + PostgreSQL schema
    └── api-client-react/       # 前端 API 请求封装 (React Query)
```

### 运行时关系

```
Browser (前端 :world-cup)
    │  所有请求带 Bearer Token
    ▼
Express API (/api/*)            ← artifacts/api-server，监听 $PORT
    ├── 行情数据 → gamma-api.polymarket.com   (公开，无需认证)
    ├── 下单     → clob.polymarket.com         (需要 L2 API Key，受地区封锁)
    └── 数据库   → Replit PostgreSQL (DATABASE_URL)
```

## 四、市场数据拉取方式

### 数据源
Polymarket 提供两套 API：

| API | 用途 | 是否需要认证 |
|-----|------|------------|
| `gamma-api.polymarket.com` | 行情数据、赔率、市场元数据 | ❌ 完全公开 |
| `polymarket.com/sports/world-cup/games` | 抓取 `__NEXT_DATA__` 获取赛事 slug 列表 | ❌ 完全公开 |
| `clob.polymarket.com` | **实际下单** | ✅ 需要 L2 API Key |

### 抓取流程（`artifacts/api-server/src/lib/polymarket.ts`）

```
1. GET polymarket.com/sports/world-cup/games
   └─ 解析 HTML 中的 <script id="__NEXT_DATA__">
      └─ 提取 props.pageProps.dehydratedState.queries
         └─ 找到 queryKey=["parentToChildEventIds"] 的条目
            └─ 过滤 key 以 "fifwc-" 开头的 slug（世界杯赛事）

2. 批量请求 gamma-api.polymarket.com/events?slug=xxx&slug=yyy...
   └─ 每批最多 20 个 slug

3. 解析每个 event：
   - 必须包含 " vs. " 格式的标题
   - 必须有 3 个市场子项（主胜 / 平局 / 客胜）
   - 从 outcomePrices 解析 YES price → 换算欧洲赔率（1 / price）
   - 从 clobTokenIds 获取下单用的 tokenId

4. 结果按 endDate 升序排列（即比赛开始时间）
5. 写入 markets_cache 表，TTL 5 分钟
```

### 置顶市场（非世界杯比赛）
- 在设置页面填入任意 Polymarket 赛事 slug
- 存入 `settings.pinnedMarketSlugs`（JSON 数组）
- 每次拉取时同样走 `gamma-api/events?slug=...`，合并后统一按日期排序

## 五、串关执行引擎（`artifacts/api-server/src/lib/parlay-engine.ts`）

### 串关生命周期
```
draft → active → won
                 lost
                 error
```

### 执行逻辑
1. `startParlayExecution(parlayId)`：draft → active，立即执行第一腿
2. 每腿执行：
   - **仿真模式**：直接标记 leg `active`，等待 cron 结算
   - **真实模式**：调用 `placePolymarketOrder()` 向 CLOB API 下单
     - 支持单注（1 个结果，全仓）
     - 支持分注（2 个结果，按 ratio 拆分下单）
     - 每腿扣除 gas 费 0.01 USDC
3. `checkAndSettleActiveLeg(parlayId)`：轮询市场结算状态
   - 胜：将赔付金额滚入下一腿本金
   - 负：整串宣告 lost
4. `pollAllActiveParlays()`：被 cron 每 60 秒调用一次

### Cron 调度（`artifacts/api-server/src/lib/cron.ts`）
- 递归 `setTimeout`（避免 `setInterval` 叠加卡死）
- Watchdog：每 5 分钟检查，若超 3 分钟无轮询则自动重启

## 七、Polymarket L2 API 认证

### 凭证存储优先级
```
环境变量（Replit Secrets）> DB settings 表
```
环境变量：`POLYMARKET_API_KEY` / `POLYMARKET_SECRET` / `POLYMARKET_PASSPHRASE` / `POLYMARKET_WALLET`

### 下单签名（`buildPolymarketSignature`）
```
signature = HMAC-SHA256(
  key     = base64decode(secret),
  message = timestamp + method + path + body
)
```
请求头：`POLY_ADDRESS` / `POLY_API_KEY` / `POLY_PASSPHRASE` / `POLY_SIGNATURE` / `POLY_TIMESTAMP`

### EIP-712 钱包绑定签名（创建 L2 Key 时用）
```js
domain  = { name: "ClobAuthDomain", version: "1", chainId: 137 }
types   = { ClobAuth: [address, timestamp(string), nonce(uint256), message] }
message = "This message attests that I control the given wallet"
// POLY_ADDRESS 必须是 EIP-55 checksum 格式
```

### 如何获取 L2 API Key（需本地开 VPN）
```bash
pip install py-clob-client
```
```python
from py_clob_client.client import ClobClient
from py_clob_client.constants import POLYGON

client = ClobClient("https://clob.polymarket.com", key="<MetaMask私钥>", chain_id=POLYGON)
resp = client.create_or_derive_api_creds()
print(resp)   # 输出 api_key / secret / passphrase
```
获得后在设置页面填入即可。

## 八、已知限制 & 当前瓶颈

### 🚫 真实下单地区封锁
- **现象**：下单时 CLOB API 返回 `403 Trading restricted in your region`
- **原因**：Polymarket 对账户注册地区（中国）或 Replit 服务器 IP 实施封锁
- **影响**：仿真模式完全不受影响；真实模式无法从 Replit 服务器直接下单

### 可能的解决方向
1. **换新钱包 + VPN 重新注册**：用非中国 IP 的新 MetaMask 账户生成 API Key
2. **本地中转代理**：在有 VPN 的本地机器上架设中转服务，下单请求经本地转发，绕开 Replit 服务器 IP
3. **继续用仿真模式**：追踪策略和赔率，不实际下单
